Biografía

Desde 1996 que cree mi primera web de manera autodidacta usando solo HTML y un bloc de notas, he dedicado todo mi tiempo libre a investigar y aprender cada una de las nuevas tecnologías que iban apareciendo aplicadas al diseño web para hacer más sofisticadas mis páginas o las páginas web que me pedían amigos y conocidos.

Para cada pagina web que me disponía a hacer buscaba siempre lo último que se usaba en diseño con la intención de aprender y de que mis páginas quedasen más "bonitas". Así, a medida que iban apareciendo, aprendí de manera autodidacta a emplear plantillas XML, DTD o XHML que junto con HTML y CSS se denominó más tarde Web 1.0. Mas adelante aprendí PHP y javascript y sus sucesivas actualizaciones y frameworks que dotaban a las páginas de dinamismo e interactividad, algo impensable años atrás y que se paso a denominar Web 2.0.

No fue hasta muchos años más tarde cuando me plantee convertir mi afición en mi profesión. Me desligue de la agencia de publicidad en la que habia trabajado toda mi vida y comence a cursar las titulaciones de técnico superior en diseño de aplicaciones web y técnico superior en administración de sistemas informáticos en red, subiendo un peldaño más en sofisticación y operatividad de las páginas web que diseñaba, una versatilidad tal que me permite hacer lo que más me gusta, pintar lienzos en la web.

Actualmente para el diseño de paginas web utilizo como base tecnológica HTML5, CSS3 (recurriendo en ocasiones a librerías como bootstrap) y el framwork AngularJS (Jquery y AJAX embebido) de javascript para el frontend y el framework PHALCON de PHP y MySQLi como gestor de bases de datos para la parte de backend. Tecnologias algunas de reciente aparición que dotan a las páginas web de una gran rapidez de proceso, una estetica atractiva y diseños 100% responsive adaptables y escalables.

Portfolio

  • SYNERGY-ITC

    Proyecto elaborado para una empresa de estados unidos dedicada a la gestion de centros educativos primarios y superiores.

    La web esta concebida como una plataforma en la cual los profesores pueden crear sus clases e incluir a los alumnos inscritos en ellas, asignarles tareas, temarios y exámenes que estos podran realizar online y gestionar e imprimer informes autogenerados en pdf con los resultados y dinamicas de sus estudiantes.

    Los alumnos por su lado recibiran notificaciones sobre las tareas y examenes y podran visionar online los temarios consistentes en videos interactivos y realizar las pruebas clasificatorias cuyos resultados recibira el profesor para su evaluación.

    debido a la conception de la web como una plataforma cais enteramente operativa, practicamente no recurro a HTML para la programacion y diseño, si no que recurri al framework CORE de PHP, jquery y AJAX con un recurso masivo de la base de datos MySQL.

  • WATERALL

    Proyecto encargado por una empresa radicada en Estados Unidos que quiere abrir una linea de negocio vendiendo un dispositivo que controlo via online el riego de los jardines.

    A traves de la web, una vez instalado el dispositivo en el domicilio, se puede dar de alta, asignarle un nombre, habilitar y gestionar los canales decidiendo un modo de funcionamiento automatico, esquematico o manual de riego, eliminar o borrar dispositivos o canales, obtener informacion metereologica de la zona donde esta instalado el dispositivo y que asi este configure un sistema de riego automatico, etc.

    La plataforma esta diseñada empleando HTML5 y CSS3 con libreria bootstrap para el FRONTEND, framework AngularJS de javascript para la funcionalidad y el framework PHALCON de PHP y una base de datos SQL para la gestion de informacion en el BACKEND.

  • CALESSE

    Proyecto actualmente en construccion para una empresa que va a iniciar un negocio de diseño y venta de mascaras tanto como complemento a indumentaria como elemento decorativo para inmuebles.

    Se trata de una web funcionalmente muy sencilla cuyo proposito en un principio esta dirigido a branding e imagen de marca, exposicion de diseños y medio de contacto. Por ello más que la funcionalidad lo que se pedia era una pagina en la que prima exclusivamente el diseño estetico.
